  • 09/2019 - 11/2019
    SimpleScalar Fault Injection Study
    Program that simulates single-event upsets (SEUs) in SimpleScalar
    For my final project in ECE 552: Advanced Computer Architecture and ECE 554: Fault Tolerant Computing Systems, I did a study of single-event upsets (SEUs) using SimpleScalar. SEUs are common due to alpha and cosmic particles. This means that mission-critical computing systems such as those in the International Space Station, or satellites/aircraft need to be properly protected with redundancy and design to tolerate soft-errors such as bit-flips.

    In this study, I analyze the probablity of an error occuring in the event of a bit-flip. This is tested with different computer architectures varying in Register Update Unit (RUU) size, fault location (register file, ALUs, instruction decoder, etc.). This simulation/study can be used in future work to determine where design companies would need to invest time/money for redundancy.
    Technologies and Skills
    C Bash Computer Architecture Fault Injection SimpleScalar
  • 07/2019
    Revit MEP Coordination Tool
    Coordination tool similar to Navisworks, but compatible with Revit
    During my Buro Happold internship one of the main things I did was working on Building Information Modeling (BIM) coordination, where I made sure that mechanical ducts, electrical conduits, pipes, etc. weren't running into each other. In the MEP industry, Autodesk's program Navisworks automates this identification of clashes and interference problems before construction. However, the limitation with Navisworks is that designers usually work through AutoCAD or Revit. A fix in clashes in Navisworks would require you to also fix it in the building's Revit model. This wastes time and money because it doubles the work.

    As a solution, collaborating with a Kayleigh Houde, the computational lead at Buro Happold, we designed a Dynamo/Python script that can detect these clashes. Because Dynamo is built into Revit, whatever clash is fixed in the model is already fixed. This prevents things like learning how to use Navisworks, switching from program to program, etc. After a group of engineers used this tool for about two weeks, it was determined that this saved the company over $11,500 and over 100 hours of coordination time.
    Technologies and Skills
    Dynamo Python Revit

    Duke-themed tower defense game built with Assembly and Verilog

  • 11/2018 - 12/2018
    Multi-Robot Path Planner
    Path planning script written in Python with robotics use-cases
    As a final project for Duke's ECE 383: Robotics and Automation, I needed to design and build a robot-path planner that would allow multiple robots to navigate a grid, such as those that navigate automated warehouses, without self nor object collision.

    This path planner was written as a Python script, and mainly utilized a breath-first-search approach. During this process, I considered different planning algorithms like Dijkstra's, but ultimately found that the best approach was breath-first-search because nodes didn't have weights. Additionally, this algorithm and path planner accounted for small hallways (in a one lane hallway, how do you determine which robot navigates through the hallway first?). This algorithm accounted for determining necessary pauses for robots in case its planned path collided with that of another.
